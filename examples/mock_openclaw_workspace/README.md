# Mock OpenClaw Workspace

A minimal OpenClaw workspace for testing `agent-bench openclaw` **without
installing OpenClaw**. The agent (`log-monitor`) monitors log streams and
triages alerts by severity — a good match for the built-in `log_alert_triage@1`
and `log_stream_monitor@1` tasks.

## Structure

```
mock_openclaw_workspace/
├── openclaw.json          # canonical agents.list config (official format)
├── workspace/
│   └── AGENTS.md          # agent system prompt (skills, behaviour, constraints)
├── cron/
│   └── jobs.json          # two example cron jobs (log scan + rate-limit watchdog)
├── tracecore_adapters/    # generated by agent-bench openclaw (gitignored)
│   └── log-monitor_adapter_agent.py
└── tracecore_export/      # generated by agent-bench openclaw-export (gitignored)
    └── log-monitor/
```

## Try it

From this directory:

```bash
# 1. Scaffold the TraceCore adapter (no OpenClaw install needed)
agent-bench openclaw --agent-id log-monitor

# 2. In your AI IDE, ask:
#    "Read tracecore_adapters/log-monitor_adapter_agent.py, workspace/AGENTS.md,
#     and cron/jobs.json. Implement act() so the agent passes log_alert_triage@1."

# 3. Run the test (red-green loop until it passes)
agent-bench openclaw --agent-id log-monitor --task log_alert_triage@1 --seed 0

# 4. Export the certified bundle (optional — for regression testing)
agent-bench openclaw-export --agent-id log-monitor
# → tracecore_export/log-monitor/  contains adapters for re-running the benchmark
#   against this agent after prompt or logic changes. Your OpenClaw agent
#   continues to run normally in OpenClaw — this is not a deployment artifact.
```

## What this actually tests

The self-contained adapter (`tracecore_adapters/log-monitor_adapter_agent.py`) is
a Python stub that mimics the agent's intended behavior — it does **not** run the
real OpenClaw agent or its model. It proves the task and harness work correctly,
not that the prompt or model behaves correctly.

The full value chain only closes when the gateway adapter is wired to a live
OpenClaw instance:

```
openclaw.json + workspace/AGENTS.md
         ↓  (system prompt fed to model)
OpenClaw gateway  (Claude Sonnet + bash / read_file tools)
         ↓  (gateway RPC: agent / agent.wait)
log-monitor_gateway_adapter_agent.py
         ↓  (translates act() ↔ gateway response)
TraceCore harness  (deterministic log files, reproducible seed)
         ↓
pass / fail  +  full action trace
```

Until the gateway adapter is wired up, the self-contained adapter is useful as:

- **A harness smoke test** — confirms the task setup and validator work correctly
- **A baseline** — documents what a correct solution looks like step-by-step
- **An AI IDE scaffold** — lets you iterate on `act()` logic before connecting the real model

## Why this agent maps well to built-in tasks

| Cron job / skill | Matching TraceCore task |
|---|---|
| Morning log scan — triage ERROR/CRITICAL | `log_alert_triage@1` |
| Streaming log pattern detection | `log_stream_monitor@1` |
| Rate-limit watchdog — 429 retry | `rate_limited_api@1` |

See `tutorials/openclaw_quickstart.md` §9 for the full capability → task table.
